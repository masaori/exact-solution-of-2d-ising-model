# 実無限を認めない数学基礎論

## 背景・動機

現代数学は ZFC 集合論を標準基盤とし、実無限（ℕ を完成した集合として扱うこと、℘(ℕ) の存在など）を前提としている。しかし：

- ℝ は「現代数学屋が欲しい機能全部盛り集合（連続性、全順序、四則 etc）」であり、個別の議論に対して過剰な抽象を持ち込んでいる可能性がある
- 非可算集合の導入が、現実世界とリンクしにくい複雑性を生んでいる
- 計算機・ML の発展により、人類の認知能力による制約（抽象化の clip）が変化しつつある

目標：実無限を認めない（あるいは最小限に留める）立場で、数学・理論物理の研究プロセスを構築するための基礎情報を整理する。

## 無限公理と実無限

ZFC の無限公理：∃x ( ∅ ∈ x ∧ ∀y (y ∈ x → y ∪ {y} ∈ x) )

- 空集合を含み、後者操作で閉じた集合の存在を保証する
- フォン・ノイマン順序数による ℕ の構成を可能にする
- ℕ は全帰納的集合の共通部分（分出公理）として得られる最小の帰納的集合

**可能無限 vs 実無限**：無限公理なしでも ∅, {∅}, {∅,{∅}}, … を一つずつ作れる（可能無限）が、それらを「まとめて一つの対象 ℕ にする」（実無限）には無限公理が必要。

## 可能無限の立場で何が失われるか

ZFC − Inf + ¬Inf（無限公理の否定を加えた体系）は PA（ペアノ算術）と本質的に同等。

### 失われるもの

1. **実数体 ℝ の構成**：℘(ℕ) や ℕ→ℕ の関数全体が必要
2. **解析学の基本定理群**：中間値の定理、最大値の定理、Bolzano-Weierstrass
3. **カントールの非可算性定理**
4. **PA を超える算術命題**：Goodstein の定理、Paris-Harrington の定理
5. **PA 自身の無矛盾性**（Gentzen は ε₀ までの超限帰納法で証明）

### 保たれるもの

- 初等整数論のほぼ全体
- 有限組合せ論

## 逆数学（Reverse Mathematics）

「各定理を証明するのに最低限どれだけの集合存在公理が必要か」を逆向きに測る研究プログラム（Friedman-Simpson）。

### Big Five

強さの昇順：

| 体系 | 強さ | 対応する数学 |
|------|------|------------|
| RCA₀ | 計算可能な数学 | 再帰的理解 |
| WKL₀ | 弱い König 補題 | 中間値の定理、Heine-Borel（閉区間） |
| ACA₀ | 算術的理解（PA 相当） | Bolzano-Weierstrass、各点収束列 |
| ATR₀ | 算術的超限再帰 | Ulm の定理、比較可能な整列順序 |
| Π¹₁-CA₀ | Π¹₁ 理解 | 可算アーベル群の構造定理 |

**意義**：ZFC で全部証明するのは「全部 String 型で書く」のと同じで、各定理の強度情報を捨てている。逆数学的アノテーションがあれば「この補題は WKL₀ で十分なので計算可能版で再利用可能」という判断ができる。

## 無限への commitment の段階的整理

下に行くほど強い実無限を認める。

### レベル 0：厳格な ultrafinitism

- **代表**：Esenin-Volpin、Nelson、Sazonov、Buss
- **認める**：具体的に到達可能な自然数のみ
- **形式体系**：bounded arithmetic (S₂, V⁰)
- **できること**：計算量理論、有限組合せ論の一部

### レベル 1：有限主義 (Hilbert)

- **代表**：Hilbert、Skolem
- **認める**：各自然数、原始再帰的に定義される関数
- **形式体系**：PRA (Primitive Recursive Arithmetic)
- **できること**：初等整数論、原始再帰の範囲の組合せ論
- **できないこと**：Goodstein の定理、PA の無矛盾性

### レベル 2：構成的算術

- **代表**：Brouwer、Heyting、Bishop
- **認める**：ℕ を完成体として（量化は構成的：witness 要求）
- **認めない**：排中律、双対否定除去、無制限の選択公理
- **形式体系**：HA (Heyting Arithmetic)

### レベル 3：型理論的構成主義 (Martin-Löf)

- **代表**：Martin-Löf、Bishop-Bridges、Aczel
- **認める**：ℕ、ℕ → ℕ を型として。modulus 付きで実数を Cauchy 列として
- **認めない**：ℕ^ℕ や ℘(ℕ) を完成集合として impredicative に量化
- **形式体系**：MLTT、CZF、HoTT（可述版）
- **できること**：構成的解析、計算可能関数論、Coq/Agda 上の形式化

### レベル 4：可述主義 (Weyl-Feferman)

- **代表**：Weyl『Das Kontinuum』、Feferman
- **認める**：ℕ、古典論理。℘(ℕ) は算術的に定義できる部分集合のみ
- **形式体系**：ACA₀、Feferman の W
- **できること**：解析の主要定理、線形代数、初等関数論
- **できないこと**：完全測度論、非可分空間

### レベル 5：非可述的構成主義

- **代表**：Aczel、Friedman、Myhill
- **認める**：℘(ℕ) を impredicative に（論理は構成的）
- **形式体系**：IZF、CZF + 強い公理
- **できること**：通常数学の大部分を構成的に再現

### レベル 6：ZFC（古典数学の標準）

### レベル 7：ZFC + 巨大基数

### 二つの直交する軸

(I) **どこまでの対象を完成体として認めるか**（個別の自然数 → ℕ → ℕ^ℕ → ℘(ℕ) → 高階集合）
(II) **どの論理を使うか**（構成的/古典的、可述的/非可述的）

## 連続性の primitive の分類

| primitive | 要求する構造 | 性質 |
|-----------|------------|------|
| ε-δ | 距離空間（ℝ 前提） | 古典解析の標準 |
| 開集合の逆像 | 位相（℘(X) 必要） | 抽象的、定量情報なし |
| フィルター/ネット | より抽象（選択公理絡み） | 一般化 |
| modulus of continuity | 定量的 | δ(ε) を明示的に与える |
| 近似保存 | 計算的 | 計算可能解析の出発点 |

**計算的視点では連続性は automatic**：計算可能関数は入力の有限部分から出力の有限部分を決めるので、必然的に連続。

## 計算可能解析（Type-Two Effectivity, TTE）

### 実数の表現

実数 x の name = 関数 φ : ℕ → ℚ s.t. ∀n, |φ(n) − x| < 2⁻ⁿ

関数 f : ℝ → ℝ の実現子 = x の name を読んで f(x) の name を書く Type-2 Turing machine

### 連続性の自動性

**定理（Kreisel-Lacombe-Shoenfield, 1959）**：計算可能関数 f : ℝ → ℝ は古典的な意味で連続。さらに effective に連続（計算可能な modulus が存在）。

**原理**：Type-2 machine が出力を n 文字書くまでに読む入力は有限個 → 入力の有限部分で出力の有限部分が決まる → Baire 空間の積位相での連続性。

### 収束（計算可能版）

Converges(x, l) := 計算可能な M : ℕ → ℕ が存在して ∀k, ∀n ≥ M(k), |x(n) − l| < 2⁻ᵏ

古典的収束に **modulus of convergence（収束速度の証拠）を添付**したもの。「収束する」だけでなく「どれくらい速く」が定義の一部。

### TTE の無限への commitment

TTE は背景に ℕ^ℕ（Baire 空間）を完成した集合として持つ。**完全な可能無限ではない**。議論の主体は計算可能対象（可算）に限定されるが、背景の宇宙は実無限。

## MLTT（Martin-Löf 型理論）での再定式化

TTE の実無限依存を避ける路線。

### 型 vs 集合

- 集合論：一階述語論理 + 述語 ∈。すべてが「集合」
- 型理論：判断（judgment）が primitive。項と型は別カテゴリ
  - A type（A は型）
  - a : A（a は型 A の項）
  - 「a : A」は判断であって命題ではない（型検査で決まる）
- **「a : A」と「a ∈ A」は異なる**：前者は文法レベルの判断、後者は真偽を問える命題

### 意味論の選択

同じ MLTT 構文に対して複数のモデルが存在：

| モデル | ℕ → ℕ の解釈 | 無限の commitment |
|--------|-------------|-----------------|
| 集合論モデル | 関数全体の集合（非可算） | 実無限 |
| realizability モデル | 計算可能関数の TM index 全体（可算） | 穏やか |
| 構文モデル | λ-項として書ける関数規則（可算、open-ended） | 可能無限的 |
| 圏論的モデル | 圏の射 | 圏依存 |

**形式体系は同じでも、無限への commitment は採用する意味論で変わる。**

### MLTT での実数

ℝ := Σ(x : ℕ → ℚ) (Π(m n : ℕ) (|x(m) − x(n)| ≤ 1/(m+1) + 1/(n+1)))

各実数項は有限の λ-項で表現される。

### 連続性原理（Continuity Principle）

CP : Π(f : ℕ^ℕ → ℕ) Π(α : ℕ^ℕ) Σ(n : ℕ) Π(β : ℕ^ℕ) (α =ₙ β → f(α) = f(β))

構成的に読むと「いかなる f, α という項が与えられても modulus n を計算する手続きがある」。ℕ^ℕ は型であり完成した集合ではない（意味論の選択による）。

## 認知制約の変化と数学方法論

### 歴史的な認知の壁の突破事例

| 年 | 事例 | 意義 |
|----|------|------|
| 1976 | 四色定理 | コンピュータによる ~1900 ケース検証 |
| 1998/2014 | ケプラー予想 / Flyspeck | 査読不能 → 完全形式化 |
| 2016 | ブール・ピタゴラス三つ組 | 200TB の証明（人間読解不能） |
| 2022 | Liquid Tensor Experiment | Scholze の結果の Lean 形式化 |
| 2023 | Tao の Lean 形式化 | AI 支援で非自明なミス発見 |
| 2026 | エルデシュ問題 #728 | AI がほぼ自律的に解決（Tao 評） |

### 抽象化とエントロピー

抽象化 = 選択肢を奪う = エントロピーを減らす。適切な抽象化とアーキテクチャは同じ原理。

- AI/形式検証の発展 → 「現実的な研究を drive するのにちょうどいいエントロピーのセットポイント」が変化
- ℝ 一発ではなく、コンテキストに応じて適切な抽象度の対象に切り替えて扱えるようになっている
- 逆数学の Big Five = 異なるエントロピーセットポイントのパレット

### 具体的な方法論の変化

1. 各定理にどの公理系を使ったかメタデータとして自動付与
2. 「最弱でどこまで弱められるか」を AI が探索
3. 同じ概念を異なる抽象度で並列に持ち、コンテキスト切り替え
4. ℝ を使った証明を「ACA₀ で十分な部分」と「ZFC が要る部分」に分解

→ dependency injection と最小権限原則の数学版

## 量子力学への適用可能性

### 構成的に再構築できる範囲

- 可分ヒルベルト空間、有界自己共役作用素、スペクトル定理の主要部分
- ユニタリ群、シュレディンガー方程式の解の存在
- 波動方程式の解は Type-2 計算可能（Weihrauch-Zhong）
- 実用上の量子力学計算はすべて正当化可能

### 障害

- **非有界作用素**（位置・運動量演算子）：ドメインを明示すれば処理可能だが手間
- **スペクトル定理と固有値**：固有値は個々には計算可能だが列挙は不可能な場合がある（Pour-El-Richards）
- **量子論理**：射影格子は分配的でなく直観主義論理ですらない

### 関連する最新研究

- "After 100 Years of Quantum Mechanics: Toward a Constructive Observation-Centered Perspective" (arXiv:2604.11814, 2026)
  - 波動関数・ハミルトニアンを primary に据えるのではなく、観測信号を primary として再構成する提案

## ML / 計算機との接続

### 理論と実装の gap

実装は完全に有限・離散（float32, bfloat16）なのに、理論解析はほぼ全部 ℝ 上。この gap を埋める試み：

1. **floating point の位置づけ**：「精度 2⁻⁵² 程度に固定された name の有限切片」（計算可能解析の枠組み）
2. **autodiff の正当化**：「線形 modulus を持つ計算可能関数」が C¹ に対応
3. **収束証明の機械化**：modulus 付き計算可能解析 → Lean 等で形式化
4. **汎化バウンドの再考**：計算可能測度論（Hoyrup-Rojas, Schroeder 等）

### 研究プロセスへの提案

- 理論証明は ℝ で書き、実装解析は計算可能解析で書く、両者は明示的な対応で接続
- 逆数学が「定理の強度」を浮き彫りにし、計算可能解析が「定理の計算的内容」を浮き彫りにする → 二つの直交する整理軸

## Sweet spot の評価

実無限を認めない研究プロセスの出発点として：

- **レベル 3（MLTT）または 4（可述主義）が sweet spot**
- レベル 3 は形式検証ツール（Coq/Agda/Lean）として実装が動いている
- レベル 0-1 は哲学的純度は高いが「形がつかない」（トレルストラ評）
- 「ℝ は数あるツールの一つに格下げ可能で、文脈次第で他のツールが優位になる」が穏当かつ射程の広い立場
