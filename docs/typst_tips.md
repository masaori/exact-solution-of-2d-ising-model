# Typst 数式記法の注意点

## 分数 (`/`) のスコープ

Typstの `/` は **隣接する1アトムにしかバインドしない**。

```typst
// NG: cosh · (K_2 / sinh) · K_2 と解釈される
cosh K_2 / sinh K_2

// OK: cosh(K_2) が分子、sinh(K_2) が分母
(cosh K_2)/(sinh K_2)
```

### 正しいパターン

```typst
// 分子・分母が複数トークンのとき → 必ず () で囲む
(a + b)/(c + d)

// display math で改行して書く場合も同じ
(cosh^2 K_2 + sinh^2 K_2)
/
(2 sinh K_2 cosh K_2)
```

### よくある間違い

```typst
// NG: ネスト分数 — 分母の中に / がある
(cosh(2 K_2^*))
/
(sinh(2 K_2^*) / 2)
// → 分母が (sinh(2K₂*)/2) のネスト分数になり読みにくい

// OK: 分母のネストを解消してから書く
(2 cosh(2 K_2^*))
/
(sinh(2 K_2^*))
```

```typst
// NG: 式の途中に裸の / がある
cosh K_2 / sinh K_2 - sinh K_2 / cosh K_2

// OK: 各分数を () で囲む
(cosh K_2)/(sinh K_2) - (sinh K_2)/(cosh K_2)
```

## インライン数式（見出し・テキスト中）での分数

見出し (`==`) やテキスト中のインライン数式 (`$...$`) では、分数表示が小さく潰れて読みにくくなる。

```typst
// NG: 見出しの中で分数
== $sqrt(A)/(B)$ について

// OK: 逆数表記にする
== $sqrt(A) dot.op B^(-1)$ について

// OK: display math にして見出しの外に出す
== $A$ と $B$ の関係
```
